# DKU 选课小助手 —— 超详细使用指南

## 这是什么？

一个帮你**浏览 DKU 课程、自动排课**的 Web 小工具。你打开浏览器访问 `http://localhost:5000` 就能用。

它能干什么：

- **浏览课程**：看这学期开了哪些课，支持 30+ 种筛选条件（学科、时间、教师、学分...）
- **查看课程详情**：每门课的时间、教室、教师、先修要求、选课名额等
- **自动排课**：你填好自己的年级、专业倾向、已修课程，它自动帮你生成一份不冲突的课表
- **毕业要求追踪**：自动检查你还需要上什么课（Common Core、语言课、PE、分布要求等）

## 第一步：安装

### 你需要先装好 Python

- 要求 **Python 3.9 或更新版本**
- 不确定有没有？打开**命令提示符**（按 `Win+R`，输入 `cmd`，回车），输入：
  ```
  python --version
  ```
  如果显示类似 `Python 3.x.x` 就说明装好了

### 安装依赖包

打开**命令提示符**，把路径切到项目目录：

```
cd D:\自己的东西\编程\选课小助手\myapp_release
```

然后安装需要的库：

```
pip install flask requests
```

> 如果你以后想用"自动获取 Cookie"功能（让程序自动打开浏览器帮你登录），还需要装：
> ```
> pip install playwright
> playwright install chromium
> ```
> 但这个不是必须的，你可以先跳过。  
> 虽然但是，如果你不会用浏览器f12自己复制cookie的话，这个就是必须的（……）

## 第二步：配置 Cookie（让程序能访问你的选课系统）

程序需要你的 DKU 登录凭证（Cookie）才能从学校的选课系统拉取课程数据。

### 方法一：手动复制（推荐新手用）

1. 用 **Chrome 或 Edge 浏览器**打开 DKU Hub：`https://dkuhub.dku.edu.cn`
2. 登录你的账号
3. 按 `F12` 打开开发者工具
4. 点击顶部的 **Network（网络）** 标签
5. 刷新一下页面（按 `F5`）
6. 在左边的请求列表里随便点一个请求
7. 在右边找到 **Request Headers** → **Cookie** 那一行
8. 把 Cookie 后面那一长串文字**全部复制**下来
9. 打开程序后，在首页的文本框里粘贴，点击"保存 Cookie"

### 方法二：用程序自动获取（需要装了 Playwright）

1. 在程序首页点击"自动获取 Cookie"按钮
2. 程序会打开一个浏览器窗口
3. 在浏览器里手动登录 DKU Hub
4. 登录成功后，程序会自动抓取 Cookie

在命令提示符里（确保路径在项目目录）：

```
cd D:\自己的东西\编程\选课小助手\myapp_release

## 第四步：使用指南

程序有 4 个主要页面：

### 1. 首页（/）—— 管理 Cookie 和同步课程数据

打开后你会看到：

- **Cookie 状态**：绿色=可用，红色=过期或无效
- **Cookie 输入框**：手动粘贴 Cookie 的地方
- **保存 Cookie 按钮**：粘贴后点这个
- **自动获取 Cookie 按钮**：用 Playwright 自动抓取
- **同步课程数据**区域：这是核心！

#### 同步课程数据的步骤：

1. 先保证 Cookie 是有效的（状态变绿）
2. 在"同步课程数据"区域：
   - 选择学期（默认是 26Fall，不用改）
   - **建议勾选"同步详情"**——这样每门课的先修要求、名额才会被拉下来
   - 点"开始同步"按钮
3. 等待同步完成，进度条会实时显示当前进度
4. 同步完成后，数据库就建好了，可以开始浏览课程和排课了

> 第一次同步可能需要 **很长很长的时间**（取决于开了多少课），因为要逐一拉取每门课的详情。（偏偏DKU的网速又很慢）
> 之后如果只是想了解一些基础信息，可以取消勾选"同步详情"，会快很多。但是这获得的数据库是不能参与排课的。

---

### 2. 课程筛选（/courses）—— 浏览和搜索课程

这个页面像一个"课程搜索引擎"：

**使用步骤：**

1. 先确保已经同步过课程数据（做过第四步第 1 节的同步）
2. 可选：在左侧筛选栏设置条件。支持的筛选条件包括：

   | 筛选条件 | 说明 | 例子 |
   |---------|------|------|
   | 学科 (Subject) | 按学科筛选 | COMPSCI, MATH, BIOL... |
   | 课号前缀 | 课号开头 | 2 (=200-level), 3 (=300-level) |
   | 课号 (Catalog Nbr) | 完整课号 | 101, 201, 301 |
   | Section | 班级号 | 001, 001L, 001R |
   | Component | 课程类型 | LEC(讲座), LAB(实验), REC(习题课) |
   | Session | 学期时段 | 7W1(前半学期), 7W2(后半学期), MNS(mini term)... |
   | 星期 | 上课日期 | M, T, W, R, F |
   | 开始/结束时间 | 时间段筛选 | 10:00-12:00 |
   | 学分 | 学分数 | 1, 2, 4 |
   | 评分方式 | 计分方式 | L(字母分), P/F(通过/不通过) |
   | 教师名 | 授课教师 | 输入名字模糊搜索 |
   | 教学楼/教室 | 上课地点 | IB, AB, CCT... |
   | 关键词 | 在课程名/描述中搜索 | "Python", "Data" |
   | 选课状态 | 名额情况 | Open(开放), Closed(满了), Waitlist(候补) |

3. 点击 **"应用筛选"** 按钮
4. 右侧会显示符合条件的课程列表
5. 点击任意课程，可以跳转到详情页
6. 也可以点 **"导出 CSV"** 把所有结果导出成表格文件

---

### 3. 课程详情（/course/<课号>）—— 深入了解一门课

点击课程列表中的任意一门课，就会进入详情页，展示：

- **基本信息**：课号、课名、学分、描述
- **时间地点**：每周哪天、几点到几点、在哪栋楼哪个教室
- **教师信息**：教师姓名和邮箱
- **选课容量**：总共多少个名额、用了多少、还剩多少
- **先修要求 (Requisite)**：上这门课需要先上过什么课。会显示**原始文本**和**解析后的逻辑树**（帮你直观理解复杂的前置条件）
- **预留座位 (Reserved Seat)**：有哪些名额是专门留给特定年级/身份的学生
- **关联课程**：如果这门课还有配套的 LAB 或 REC，会在这里显示
- **手动刷新按钮**：如果觉得数据过时了，可以单独刷新这门课

---

### 4. 自动排课（/schedule）—— 核心功能！

这是本程序最强大的功能：输入你的信息，自动生成一份**不冲突、满足毕业要求**的课表。

#### 使用步骤：

**第一步：填写个人信息**

- **年级 (Year Level)**：Y1 / Y2 / Y3 / Y4
- **Class Of**：你的毕业年份，比如 `2029`
- **身份 (Student Identity)**：
  - `chinese` — 中国学生（会安排 EAP 英语课）
  - `international` — 国际生（会安排 CSL 中文课）
  - `heritage` — 华裔学生（会安排 Heritage 中文课）
  - `other` — 其他情况

**第二步：选择已修课程**

有两种方式：
- **自动同步**：点按钮从 DKU 自动拉取你历史上过的课
- **手动输入**：手工填写每行一个课号，比如 `MATH 101`, `CCORE 101`

**第三步：选择目标课程（可选）**

- **"必须包含"的课**：你指定一定要排进去的课（比如你想上的某门专业课），每行一个课号+section
- **"考虑包含"的课**：你感兴趣的课，算法会尽量排进去但不保证

**第四步：设置排课约束**

- **学分范围**：最少和最多多少学分（比如 16-20）
- **Session 学分**：S1/S2 各自最多学分（防止半个学期太累）
- **PE 数量**：这学期想排几门体育课
- **大一 S1 限制**：大一上学期不能超 2 门 100-level S1 课

**第五步：设置软偏好**

这些不是硬性要求，算法会综合考虑来给课表打分：

| 偏好 | 说明 |
|------|------|
| 专业倾向 | 输入你的 major，算法会优先排这个专业的课 |
| 小课偏好 | 越大越喜欢 2 分的小课（凑学分用） |
| 风险规避 | 越大越倾向于选"名额充足的课"（降低选课被踢风险） |
| 紧凑/松散 | 越大越喜欢课排得紧（少空档），越小越喜欢课散开 |
| 早八厌恶 | 越大越讨厌早上 8 点的课 |
| 晚课厌恶 | 越大越讨厌晚上的课 |
| 教授偏好 | 有特别想跟/不想跟的老师可以写这里 |
| 属性偏好 | 对某些课程属性（如 W=写作课, QR=量化推理）的倾向 |

**第六步：设置运行参数**

- **尝试次数**：算法随机尝试多少次来生成课表（越大越可能找到最优解，但也更慢）。建议 5000-20000
- **目标方案数**：最终输出几个不同的课表供你选择。建议 3-5
- **随机种子**：固定随机种子可以让结果可复现（留空=每次不同）

**第七步：点"开始排课"！**

排课过程中的算法逻辑：

1. 算法会从数据库里读取所有可选的课程
2. 自动识别**毕业要求**需要什么课（比如大一上学期要上 CCORE 101 + EAP + W 写作课）
3. 自动检查**先修关系**：确保按 Session 顺序，每门课的前置课都在本 Session 之前修过
4. 自动**避开时间冲突**：不会把两门时间重叠的课排到一起
5. 随机组合 → 打分排序 → 输出 Top-N 个最佳课表

**结果展示：**

排完后会显示：
- 排出的课表数量
- 每个课表的总分和详细评分
- **可视化 Timeline**：一张网格图，横轴是星期一到五，纵轴是时间 8:00-20:00，每门课是一个色块
- 每门课的详细信息（时间、地点、教师）
- 课表的毕业要求完成情况

## 常见问题

### Q: 启动报错 `ModuleNotFoundError: No module named 'flask'`
**A:** 说明没装 Flask。运行 `pip install flask`。

### Q: Cookie 一直显示红色/过期
**A:** DKU Hub 的 Cookie 一般有效期只有几小时。重新登录 DKU Hub，重新复制 Cookie 即可。

### Q: 同步课程数据时卡住不动了
**A:** 可能是网络问题或 Cookie 中途过期。关闭程序，重新获取 Cookie，重新同步。另外，同步课程数据的时候慢是正常现象，还请稍安勿躁。

### Q: 排课排不出来（0 个方案）
**A:** 可能的原因：
1. 目标学分太高或太低，没有课程组合能满足
2. "必须包含"的课之间有不可解决的时间冲突
3. 先修要求不满足（你要选的课需要先修课，但你还没上过）
4. 尝试调大"尝试次数"参数

### Q: 数据库在哪里？怎么清理？
**A:** 数据库文件在 `databases/` 目录下，文件名格式为 `UGRD-2268-26FALL-日期时间.db`。你可以删除旧的 .db 文件来清理空间（不影响程序运行，下次同步会生成新的）。

### Q: 这个程序安全吗？我的 DKU 密码会不会泄露？
**A:** 程序使用的是 Cookie 认证，**不获取也不存储你的密码**。Cookie 保存在你本地的 `dku_cookie.txt` 文件里（已被 .gitignore 忽略，不会上传到 Git）。所有数据都在你的电脑本地处理。

└── databases/             ← SQLite 数据库存放处（初始为空）
```

## 高级功能

### 多学期支持
在同步页面可以选择同步不止一个学期（比如同时同步 26Fall 和 26Spring），之后在排课时可以选择不同的学期。

### CSV 导出
在课程筛选页面可以导出当前搜索结果到 CSV 文件，方便用 Excel 打开分析。

### Requisite Eval Tree
在课程详情页面，复杂的先修条件会自动被解析成一棵逻辑树，用 AND/OR 节点可视化展示，帮你理解"到底要上过哪些课才能选这门课"。
